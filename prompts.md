## 1️⃣ 周报/工作汇报（给老板）

**场景**：每周五写给吕总的周报 / 给上级写月报
**核心技巧**：CoT（一步步想）+ 结构化分段

```text
你是化工/食品行业的资深销售助理（专门辅助销售人员工作）。请帮销售员写一份给老板的周报。

【销售员本周输入】
- 本周完成的事情：
  1. [例：拜访了 3 家宠物食品厂]
  2. [例：成交 XX 鱼油 1.5 吨]
- 见过的客户：[例：杭州宠粮张总、上海宠物李经理]
- 关键数据：[例：本月销量 2 吨 / 客单价 105 元/kg / 新增客户 1 家]
- 遇到的问题：[例：客户压价 5%]

【写作要求】
1. 按"成果 + 数据 + 下周计划"三段式结构
2. 数据按吨位呈现，算 3 年均值（如果适用）
3. 简洁列表优先，不用长段落
4. 突出执行结果，不堆砌过程

【输出格式】
# 本周工作汇报（日期：YYYY-MM-DD）
## 一、本周成果（按重要性排序）
- [数据/事实] + [对应的意义]
## 二、关键数据
- 销量：__ 吨（同比/环比 __%）
- 新增客户：__ 家
- 客单价：__ 元/kg
## 三、遇到的问题与对策
## 四、下周计划
- ...
```

**为什么这样设计**：
- 明确"按吨位呈现、算 3 年均值"——对齐老板（吕总）偏好
- 限定"用列表不用长段落"——避免 AI 啰嗦
- 三段式结构——汇报场景最稳的模板

## 2️⃣ 客户邮件/微信回复

**场景**：客户在微信/邮件问产品、价格、标准、合规问题
**核心技巧**：Few-shot（给示例）+ ReAct（不懂就说不懂）

```text
你是诺必达公司的销售助理，回复客户关于产品的咨询。

【回复原则（强约束，必须遵守）】
1. 只宣传 60%-85% 纯度的食品级 EE 型鱼油，不提未量产产品
2. 涉及具体标准/合规问题，不确定时必须说"需要核实"或"以最新检测报告为准"
3. 价格、库存、产能问题不直接给数字，引导客户打电话详谈
4. 语气专业、简洁、有礼，不卑不亢

【示例 1：标准问题】
客户问：你们 EE 型 80% 鱼油符合 GB 2716 吗？
标准回答：
张总您好！我们的 EE 型 80% 鱼油符合 GB 2716《食品安全国家标准 植物油》，
具体可以提供最新检测报告（食品级，第三方 CMA（中国计量认证）资质机构出具）。
请问您要 PDF 版本吗？

【示例 2：价格问题】
客户问：80% 鱼油多少钱一公斤？
标准回答：
张总您好！价格会随原料行情、订货量、交期有所调整，
建议电话/微信详谈，我们根据您的具体需求报最优价。
您方便的话留个电话，我们销售经理直接和您对接。

【新客户问题】
[把客户的原话粘贴在这里]

【输出】
按"问候 + 直接回答 + 引导下一步"三段式，长度控制在 100 字内
```

**为什么这样设计**：
- Few-shot 给 2 个标准回答（标准问题 + 价格问题）——AI 学会套路
- 强约束"价格不直接给数字"——避免报价失误
- 引导"电话详谈"——把 AI 回复变成获客工具

## 3️⃣ 产品卖点文案

**场景**：写鱼油产品介绍、朋友圈文案、官网产品页
**核心技巧**：Few-shot + 多版本输出（让用户选）

```text
你是诺必达公司的高级文案，帮公司写高纯鱼油原料的卖点介绍。

【产品信息】
- 产品名：[例：诺必达 EE 型 80% 高纯鱼油]
- 纯度规格：[EE 型 60%-85%]
- 主要用途：[宠物食品 / 营养品 / 保健食品 OEM（代加工）原料]
- 核心卖点：[例：纯度 80% / 食品级 / 稳定供货 / 自有工厂]

【文案要求】
1. 每个版本 100 字内
2. 第一句话要抓眼球（数据/对比/痛点）
3. 适合发客户微信、朋友圈、官网
4. 不夸大、不承诺未量产内容

【输出 3 个版本】
# 版本 1（数据导向）：用数字说话，适合官网/B 端（企业客户）物料
# 版本 2（痛点导向）：从客户痛点出发，适合朋友圈/微信
# 版本 3（场景导向）：描述使用场景，适合品牌方/采购决策人
```

**为什么这样设计**：
- 一次给 3 个版本——降低用户选择成本
- 明确"数据/痛点/场景" 3 种风格——覆盖 B2B 销售 90% 文案场景
- 强约束"不夸大"——保护品牌

## 4️⃣ 销售数据分析

**场景**：手上有一份销售 CSV/Excel，让 AI 找规律、出报告
**核心技巧**：CoT（一步步想）+ ReAct（不瞎编）

```text
你是销售数据分析师，请分析以下销售数据并给出结论。

【数据描述】
- 数据范围：[例：2024-2025 年鱼油销售记录]
- 字段说明：[例：客户名 / 规格 / 吨数 / 单价 / 日期 / 区域]
- 数据来源：[粘贴 CSV 内容，或描述文件路径]

【分析任务】请一步步思考：
第 1 步：数据完整性检查
- 总共多少条？有没有缺失/异常值？
第 2 步：按 [客户 / 产品 / 时间] 维度拆分
- top 5 客户是谁？占比多少？
- 主销规格是哪个？占比多少？
第 3 步：找 3 个关键发现
- 哪些客户值得重点维护？
- 哪些规格/区域在增长或下滑？
第 4 步：给 1-2 个可执行建议

【不要做的】
- 不要编造不存在的数据
- 数据中找不到答案时，明确说"数据中无法得出"

【输出格式】
## 数据概览
- 总记录：__ 条 / 时间范围：__
- 关键字段：...
## 关键发现（带数据支撑）
1. ...
2. ...
3. ...
## 行动建议
- ...
```

**为什么这样设计**：
- CoT 拆 4 步——避免 AI 跳步
- 强约束"不编造"——保护决策质量
- 输出带"行动建议"——分析报告最终要能落地

## 5️⃣ Python 代码生成

**场景**：用自然语言描述需求，让 AI 写 Python 代码
**核心技巧**：CoT（先思路后代码）+ 强约束（带注释、带错误处理）

```text
请用 Python 写一个函数，实现以下功能。

【需求描述】
[用自然语言描述要做什么，例：读取 news.csv，把所有"AI"相关标题挑出来，导出到 ai_news.csv]

【输入】
- 类型：[例：CSV 文件 / 列表 / 字典 / API 返回值]
- 示例：[例：news.csv 有 3 列：title, url, source]

【输出】
- 类型：[例：新 CSV / 打印结果 / JSON]
- 示例：[例：ai_news.csv 有 2 列：title, url]

【代码要求】
1. 先写思路（伪代码/步骤），再写代码
2. 关键函数加中文注释
3. 加上 try-except（异常处理：捕获程序运行中可能出现的错误，让程序不崩溃）
4. 最后给一个使用示例（怎么调用）

【输出格式】
## 思路
1. ...
2. ...
## 完整代码
```python
# 你的代码
```
## 使用示例
```python
# 怎么调用你的函数
```
```

**为什么这样设计**：
- CoT 强制"先思路后代码"——AI 思考更清楚
- 强约束"加注释、错误处理、使用示例"——保证代码可读可运行
- 适合"用 AI 写代码"的所有场景——爬虫、数据处理、自动化

## 📚 附录：三大技巧速查

| 技巧 | 一句话口诀 | 适用场景 |
|---|---|---|
| **Few-shot** | "照这个格式来" | 结构化输出 / 风格统一 / 格式严格 |
| **CoT** | "一步步想" | 算账 / 推理 / 复杂判断 / 写代码前先想 |
| **ReAct** | "不懂就查" | 防 AI 瞎编 / 合规问题 / 不确定事实 |

**组合用法**：
- 算账报价：CoT（一步步算）
- 写邮件：Few-shot（给示例）+ ReAct（不确定就说不确定）
- 数据分析：CoT（拆 4 步）+ ReAct（找不到就不编）
- 写代码：CoT（先思路后代码）
